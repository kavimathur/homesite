          Generic Openings 
          
        

        
Greetings for non-prompt Home-page calls. The iOS client picks one at random per call; use {name} in the opening line to substitute the user's first name (those rows are only used when we have a name on file). Time-of-day rows are only eligible when the user's local clock matches the bucket (morning 5:00am–11:59am, afternoon 12:00pm–5:29pm, evening 5:30pm–12:29am, late night 12:30am–4:59am); "Any time" rows are always eligible.


        

          

            
            
          

          

            
            
          

          
          

            
            
            
          

        


        

      

    

  


  

    

      
      
      
    

    

    

      

        
Production Realtime Prompt

        
This edits the base prompt used by the production get-realtime-config edge function. Personalized memory layers are still appended dynamically for each user.

        

          
          

            
            
          

        

      

    

  


  

    

      
      
      
    

    

    

      

        
Dev Realtime Prompt DEV ONLY

        
Only used by debug builds of the iOS app (i.e. when running from Xcode). TestFlight and App Store builds always use the production prompt. Leave this blank to make debug builds also use the production prompt.

        

          
          

            
            
          

        

        

          
          
        

      

    

  


  

    

      
      
      
    

    

    

      

        
Onboarding Intro Prompt

        
Scripted prompt used during the voice-onboarding intro call (Turns 1–7). Served to the realtime model with NO memory injection. The complete_intro_call tool is attached automatically — this textarea controls only the conversational instructions.

        

          
          

            
            
          

        

      

    

  


  

    

      
      
      
    

    

    

      

        
Onboarding First Entry Prompt

        
Base prompt used during the user's first journal entry inside onboarding (3-minute call). Independent of the production base prompt. Memory layers are NOT injected. The user's selected journal prompt (title + opening line) is appended as a per-call suffix client-side.

        

          
          

            
            
          

        

      

    

  


  

    

      
      
      
    

    

    

      
Per-prompt addendums appended to the realtime base prompt when a new user starts their first journaling session, depending on which prompt they pick. Each is a verbatim opening line plus steering. Use [name] for the user's first name (substituted in-app). Each prompt has a Production copy (shipped users) and a Dev copy (DEBUG builds, matching the dev realtime base prompt) — a blank Dev box falls back to Production. Blanking Production makes the app fall back to a generic opener. The "no prompt, just talk" option has no addendum. Edits save automatically.
